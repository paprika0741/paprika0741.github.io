1.为什么要并行计算

如何编写

  • 任务并行 task-parallelism

    分配任务到各个核上

  • 数据并行 data-parallelism
    数据分配给各个核，执行大致相同操作

    协调的问题

  • 通信communication
    一个或多个核将自己的部分结果发送给其他核

  • 负载均衡 load balancing
    工作量差不多

  • 同步synchronization

    任务有先后次序

  • 共享内存系统 Shared_memory

    Pthreads和OpenMP
    各个核都能共享计算机的内存，检测和共享更新内存中的数据来协调各个核。

  • 分布式内存系统 Distributed-memory

    MPI，提供发送消息的机制
    各个核都拥有自己的私有内存，
    核之间的通信是显式的

并发 并行 分布式

并发计算： concurrent computing

分布式计算： distributed computing

分布式：distributed
一个程序与其他程序紧密协作来解决某个问题(进程协作)
松耦合，多个计算机执行

并行：parallel
一个程序通过多个任务紧密协作来解决某个问题
紧耦合，并行往往同时在多个核执行多个任务

并发 ：concurrent

​ 一个程序的多个任务在同一时段同时执行。

并行程序和分布式程序都是并发的
某些操作系统也是并发的,即使只有一个核,多任务能在同一时段里同时执行

2、并行硬件和并行软件

2.1冯诺依曼结构

冯诺依曼瓶颈：

主存和CPU之间的分离。
互联结构限制了指令和数据访问的速率。
程序运行所需要的大部分数据和指令被有效地与CPU隔离开。
CPU执行指令速度比取指令快很多倍

进程

An instance of a computer program that is being executed


进程的组成：

  • The executable machine language program.（程序段)
  • A block of memory（分配内存空间）
  • Descriptors of resources the OS has allocated to the process（资源分配清单）
  • Security information（安全信息）
  • Information about the state of the process（进程描述信息/控制信息）

多任务 Multitasking

单处理机，时间片轮转

线程

程序划分为多个大致独立的任务，某个任务阻塞的时候执行其他任务。
线程切换比进程快。
线程阻塞进程并不阻塞

forking and joining

Cahce

局部性 (locality)：
即在访问一个内存位置之后，程序通常会在不久的将来（时间局部性, temporal locality）访问一个附近的位置(空间局部性, spatial locality)

虚拟内存：

主存最为辅存的缓存.内存中只有进程活跃的部分,利用时间局部性和空间局部性

有些进程可能比内存还要大 — 部分加载 不连续分配 交换空间

缺点:访问主存时间加倍

TLB
缓存一部分标签页表条目,利用时间局部性和空间局部性,大幅减少访问主存时间

指令级并行

指令级并行性(ILP)：
多个处理器组件或功能单元同时执行指令来提高处理器性能

multiple processor components or functional units simultaneouslyexecuting instructions

主要方法有：

  • 流水线 (pipelining) ：
    将功能分为多个单独的硬件或者功能单元，
    将功能单元分阶段安排

  • 多发射 (multiple issue) ：
    可以同时启动多个指令
    复制功能单元来同时执行程序中的不同指令

    静态多发射：编译时调度 功能单元
    动态多发射：运行时调度 功能单元

    超标量：superscalar
    支持动态多发射的处理器

    预测：speculation
    为了利用多发射，系统必须能够找出能够同时执行的指令。
    编译器/处理器对一条指令进行猜测，然后在猜测的基础上执行代码。

    (如果猜错了,go back)

硬件多线程

硬件多线程 (Hardware multithreading)
当前正在执行的任务处于停滞（stalled）状态时，系统可以继续执行其他有用的任务。
系统必须支持线程间的快速切换

ILP很难使用，尤其是对于存在依赖语句的程序而言，比如通过循环求斐波那契数列。

此时则可以通过线程级并行来提高运行效率。

  • 线程级并行性 (Thread-level parallelism, TLP)
    通过同时执行不同的线程来提供并行性

    提供了比ILP更粗粒度的并行性，也就是说，同时执行的程序单元(线程)比细粒度的单元(单个指令)更大或更粗。

    • 细粒度多线程 (Fine-grained multithreading )
      处理器在每条指令之后在线程之间切换，跳过停止的线程。
      优点：避免阻塞而导致机器时间的浪费
      缺点： 执行很长一段指令的线程，在执行每条指令的时候都需要等待
      • 同步多线程 （Simultaneous multithreading, SMT）
        是细粒度多线程的变体。
        通过允许多个线程同时使用多个功能单元来利用超标量处理器的性能。
    • 粗粒度多线程 (Coarse-grained multithreading)
      只切换需要等待较长时间才能完成操作（如从主存中加载）而被阻塞的线程
      优点：不需要线程间的立即切换
      缺点：处理器还是可能在短阻塞时空闲，线程间的切换也还是会导致延迟。

并行硬件

Flynn分类法

SISD

典型的冯诺依曼系统

Single instruction stream Single data stream

一次执行一条指令 一次存取一个数据项

SIMD

Single Instruction Multiple Data
并行系统

  • 多个数据执行相同的指令

    通过在处理器之间划分数据来实现并行性

    (数据并行)

  • 适合并行化操作大型数据数组的简单循环

  • 一个抽象的SIMD系统可以被认为仅有一个控制单元，但有多个ALU。

Drawbacks:
一条指令从控制单元广播到ALU，每个ALU要么将指令应用到当前数据项上，要么它是空闲的

ALU 没有指令存储

处理大型数据并行问题时非常有效，
但是SIMD系统在处理其他类型的并行问题时往往做得不是很好。

Vector Processors

传统CPU处理单独数据单元/标量
向量处理器处理一组向量

向量寄存器(Vector registers)
存储多个操作数组成的向量, 可以同时对其内容操作

Vectorized and pipelined functional units
向量化和流水化功能单元
向量中的每个元素需要相同的操作(SIMD)

向量指令(Vector instructions)
操作向量而不是标量
只需一次加载 一操作完成对 vector_length的数据块操作

交叉存储器(Interleaved memory)
内存系统分为多个部分,每个bank可以独立访问
访问同一个”bank”需要延迟,访问一个接着另一个没有延迟

将向量的元素分布在多个组中，从而减少或消除加载/存储连续元素的延迟

步长式存储器访问和硬件散射/聚集
Strided memory access and hardware scatter/gather
跨步内存访问和硬件分散/收集

程序访问位于固定间隔的向量元素
散射/聚集:对无规律间隔数据读(聚集) 写(散射)
向量系统提供特殊的硬件加速

pros cons
fast / easy to use 不处理不规则的数据结构以及其他并行架构
向量化编译器擅长识别要利用的代码 可扩展性差:处理更大问题的能力非常有限
高内存带宽
编译器还可以提供有关无法矢量化的代码的信息,帮助程序员重新评估代码
使用缓存行中的每个项目
GPU

图形处理单元
严重依赖硬件多线程

GPU 通常可以通过使用 SIMD 并行来优化性能
每个GPU处理核引入大量ALU
需要处理大量数据的线程维持ALU的忙碌,

SIMD: Vector processor && Graphics Processing Units (GPU)

MIMD

Multiple Instruction， Multiple Dara

  • 多个指令流在多个数据流上操作

  • MIMD系统通常由一组完全独立的处理单元或核组成，每个处理单元或核都有自己的控制单元和自己的ALU

  • 与SIMD系统不同，MIMD系统通常是异步的。各个处理器按照自己的节奏运行

  • 两种主要类型的MIMD系统：
    共享内存系统：
    一组自治的处理器通过互联网络与内存系统相互连接，每个处理器能够访问每个内存区域。
    处理器通过访问共享数据结构隐式通信

    ​

    | | UMA一致内存访问系统 | NUMA非一致内存访问系统 |
    | —— | —————————————————————————————— | —————————————————————————————— |
    | | | |
    | | 所有处理器之间连接到主存 | 每个处理器连接一块内存,处理器内置硬件可以访问内存的其他块 |
    | | 访问任何区域时间相同 | 访问之间相连的区域快很多 |

分布式内存系统：
每个处理器都要自己私有的内存空间
处理器-内存对之间通过互联网络互相通信
处理器之间通过发送消息或者使用特殊的函数访问其他处理器的内存

​

集群:Clusters

使用最广泛的分布式内存系统

集群是一组相互独立的、通过高速网络互联的计算机，它们构成了一个组，并以单一系统的模式加以管理。一个客户与集群相互作用时，集群像是一个独立的服务器。

集群的节点是由通信网络连接的单个计算单元

一组商品化系统组成，通过商品化连接

SPMD

SPMD 程序由单个可执行文件组成，通过使用条件分支，它可以像多个不同的程序一样运行

Interconnection networks

共享内存互联网络

  • bus interconnect
    并行通信线路与一些硬件构成的集合，控制了对总线（bus）的访问

    • 优点
      低成本 灵活; 多个设备可以连接到一个总线，几乎没有额外的成本
    • 缺点
      由于通信线是共享的，随着连接到总线的设备数量的增加，对总线的使用发生争用的可能性增加，总线的预期性能下降

  • 交换互联

    crossbar 交换开关矩阵
    使用交换机来控制被连接设备之间的数据路由

    这些线路是双向通信链路，正方形一部分代表的是核，一部分代表了存储模块，圆形是交换器

    • 优点:
      允许不同设备之间同时通信，因此它们比总线快得多。
    • 缺点:
      交换器和链路的成本相对较高。

    分布式内存互联网络

    • 直接互联
      每个交换器直接连接到处理器-存储器对，并且交换器之间彼此连接(“交换器”圆圈,”处理器-存储器对”方块)

      • 圆环(ring)
        优于简单的总线，允许多个同时通信。
        交换器支持3个链路

      • 环面 (toroidal mesh)
        比环昂贵，因为交换器更加复杂
        交换器支持5个链路

        • 如果p个处理器
          环中链路:2p
          (首先处理器和交换机相连有p个,p个交换机成环,这个圈上有p个,总共2p)

          环面链路: 3p
          (假设有$p=mn$个处理器，首先p个，然后横向的有$nm$ 纵向也是$m*n$，总共3p)

    • 间接互联
      交换器不一定与处理器直接连接
      由一些单向连接和一组处理器组成

      每个处理器有输入链路和输出链路

等分宽度(bisection width)

(两个等分之间最少链路数)

衡量“同时通信的链路数目”或者“连接性”的一个标准

方法①

系统等分两部分，每部分一半结点，同时发送通信，给出一个“最坏情况”的估计

方法②
去除最少链路从而将结点两等分

​

正方形二维环面网络$2\sqrt{p} $

等分带宽 Bisection bandwidth

带宽 : 链路传输数据的速率

等分带宽 : 网络质量的衡量标准. 两个等分之间的带宽（最坏情况）

$eg:$ 环每条链路式p，等分是 p+p=2p

全连接网络

每个交换机都直接连接到其他每个交换机
等分宽度 = $\frac{p^2}{4}$
一边一半$\frac{p}{2}$ ,每个都和另一边有变，总共$\frac{p}{2}\times\frac{p}{2}=\frac{p^2}{4}$
总的链路: 交换机互联$\frac{p^2}{2}-\frac{p }{2}$每个交换机连接一个处理器-存储器对,+p,总共$\frac{p^2}{2}+\frac{p }{2}$

2.3.4cache一致性

多核系统，各个核cache存储相同变量的副本，当一个处理器共享cache中该变量的副本，其他处理器也应该更新

​
方法①： 监听cache一致性
更新时广播通知其他核包含x的cache已经更新

(其实是更新cache行)

方法②：基于目录的cache一致性协议
广播对于大型系统性能下降，不可扩展。
通过一个目录的数据结构存储每个缓存行的状态。
目录是分布式的,每个核-内存对存储一部分目录
当变量需要更新，会查询目录，将包含变量的行高速缓存标记非法
优点:当一个Cache变量更新,只需要与存储这个变量的核交涉
缺点:需要额外的空间存储

伪共享

缓存系统中是以缓存行（cache line）为单位存储的，对缓存行中的单个变量进行修改了，导致整个缓存行数据也就失效了，并且，会导致其他CPU的含有被改动的共享变量的缓存行也失效了，就是所谓的伪共享问题

​

过多不必要访问主存，降低存储器性能
解决：线程/进程临时存储数据，再把临时存储的数据更新到共享存储降低共享带来的影响

2.6性能

2.6.1 加速比/效率

Speedup加速比: $S = \frac{T_s}{T_p}$

线性加速比: $S = p \ \ E = 1 $

效率: $E = \frac{S}{p}$

问题规模变大,加速比和效率增加;
规模小,加速比和效率降低

开销的影响
$T_p = \frac{T_s}{p} + T_{overhead}$

2.6.2阿姆达尔定律

除非一个串行串行的执行几乎全部都并行化,
否则,不管多少核,加速比都会是受限的

:smile:

2.6.3可扩展性

增加进程/线程的个数,以相应倍率增加问题的规模才能使效率保持不变

  • 强可扩展
    增加进程/线程的个数,不增加问题的规模,可以维持固定的速率

  • 弱可扩展

    增加进程/线程的个数,以相同的倍率增加问题的规模才能使效率保持不变.
