Pthread

共享内存式编程

线程对象pthread_t

​

  • 不透明对象（Opaque）

  • 线程调度是由操作系统决定的，没有参数指定在哪个核上运行

  • 如果线程数大于核数，多个线程可能运行在一个核上

创建 pthread_create()

第一个参数 指针，指向pthread_t对象，注意是在调用create函数前就创建分配内存空间的

第二个参数 不用NULL

第三个参数 表示将要运行的函数void thread_function(void args_p)

第四个参数 传给函数的参数 显示声明(void *) 实际上可能是int,因为C语言中void可以转换任意指针类型

注意：

  • 指令传thread分配编号只是为了方便
  • 运行的函数不一定相同

void thread_function(void args_p)

Void* can be cast to any pointer type in C

So args_pcan point to a list containing one or more values needed by thread_function

the return value of thread_functioncan point to a list of one or more values

常用： 唯一参数int型参数rank，表示线程的编号

把循环变量thread 转换成void* 类型

线程函数中 将这个参数转换成long型

停止线程

临界区Critical sections

先到者会被后来者重写

多个线程访问共享文件/变量，如果至少一个是更新操作，那么这些访问会导致某种错误

忙等待Busy-Waiting

​

  • 一个flag表示允许进入临界区的线程号
    离开临界区时修改flag，让下一个进入

  • 最小化执行临界区的次数

Mutexes互斥量

  • 一种特殊的变量
  • 限制每次只有一个线程进入临界区，保证一个线程独享临界区

数据类型： pthread_mutex_t ， 使用前初始化

初始化


pthread_mutex_init(&mutex, NULL);

lock/unlock

In order to gain access to a critical section a thread calls 获得临界区访问权

会使线程等待，直到临界区中没有其他线程再进入

When a thread is finished executing the code in a critical section, it should call 退出临界区

通知系统该线程已经完成了对临界区的操作

调用lock会等待，直到没有其他线程加入临界区。

第一个调用lock的线程会为临界区“锁门”，其他线程要加入临界区也要先调用lock，都会等待第一个线程离开临界区。

只有第一个线程unlock，阻塞的才会选一个进入临界区，直到所有的都完成临界区的操作

先把不会冲突的算了，增加并行代码，尽量减少需要串行的，增加效率

信号量

一种特殊的unsigned

如果二元信号做互斥量,初始化1, 开锁状态

要保护的临界区前调用sem_wait, 如果信号量0,阻塞,不为0,减1进入临界区

信号量和互斥量最大区别： 信号量没有个体拥有权，主线程将所有的信号量初始化为0，即加锁，其他线程都能对任何信号量调用sem_post和sem_wait函数

init: 第一个参数可以是数组的首地址，初始化多个信号量

destroy同理

sem_wait: 如果0,等待; 如果1,减1进入临界区

post:+1 ， 使得在sem_wait 阻塞其他线程能够继续运行

使用信号量保护临界区

①一个全局变量的信号量sem, main函数中初始化1，开锁状态

②因为sem初始化1，第一个到达sem_wait(&sem) 的线程能够进入临界区，并减一，

加锁，其他的线程都阻塞在 sem_wait(&sem)这里

③第一个线程完成临界区，调用sem_post(&sem)， sem加一，

使得在sem_wait 阻塞其他线程能够继续运行

④下一个线程进入临界区，减一…..循环处理上述过程

缓存一致性

缓存基本单位是缓存行/缓存块：两个线程可能在访问不同的内存单元，若在同一个缓存行，缓存一致性认为在访问同一个内存单元。这样如果一个更新了值，之后其他的线程读这个块，需要从内存获取

缓存一致性硬件强迫多个线程看起来好像在共享一个内存单元（伪共享） ， 严重降低共享内存程序的性能

线程安全性

代码块能够被多个线程执行而不引起问题—安全的

某些C语言函数声明static变量在两次调用之间缓存数据。多个线程调用该函数会有错误，以为线程之间共享静态存储，一个线程会覆盖另一个线程的数据
