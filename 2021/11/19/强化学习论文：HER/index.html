Hindsight Experience Replay

Sparse reward problem

  • need for complicated reward engineering

    requires both RL expertise and domain-specific knowledge
    we must know something about how to solve the problem in order to properly shape the reward.

  • Another danger is that once we engineered the reward, we are no longer directly optimizing for the metric we are really interested in, but instead optimizing a proxy that we hope will make the learning process easier.
    一旦我们设计了奖励，我们就不再直接针对我们真正感兴趣的指标进行优化，而是优化一个我们希望能让学习过程更容易的代理

  • sometimes even lead to unexpected and unwanted behavior, that might necessitate a frequent fine tuning of the engineered reward in order to get it right.

Hindsight Experience Replay:

  • allows sample-efficient learning from rewards which are sparse and binary

  • avoid the need for complicated reward engineering, learn from unshaped reward signals, e.g. a binary signal indicating successful task completion.

  • 可以结合任意的 off-policy 函数

  • multiple goals
    eg. 实现系统的每个状态都可以被视为一个单独的目标

pivotal idea :
人类相较于代码中的Agent，有一个明显的长处就是在无法得到明确的反馈的情况下，依然可以积累一定的经验，而Agent却无法获得任何的收获。那么我们是否可以把这部分经验也告诉Agent

replay each episode with a different goal than the one the agent was trying to achieve, e.g. one of the goals which was achieved in the episode.

例子：

踢球 射击目标偏右 RL算法学到的很少，只知道所执行的动作序列不会导致一个成功的射击

但是，有可能得出另一个结论，即如果把网在更右边，这一系列行动就会成功

如果换个角度的用相同的动作或者将球网移动一些位置就可以击球得分。

HER lets an agent learn from undesired outcomes and tackles the problem of sparse rewards in Reinforcement Learning

使智能体从没达到的结果中去学习，解决了强化学习中稀疏奖励的问题。

Background

  • Deep Q-Networks (DQN)
    discrete action spaces

  • Deep Deterministic Policy Gradients (DDPG)
    continuous action spaces

  • Universal Value Function Approximators (UVFA)

    • an extension of DQN to the setup where there is more than one goal we may try to achieve
      DQN的扩展， 目标不止一个

    • Every episode starts with sampling a state-goal pair from some distribution

      p(s0, g). The goal stays fixed for the whole episode.

      每个episode 采样目标状态对，一个episode中的目标固定

    • policy: $\pi: \mathcal{S} \times \mathcal{G} \rightarrow \mathcal{A}$

      reward: $r_{t}=r_{g}\left(s_{t}, a_{t}\right) .$

      Q-function: $Q^{\pi}\left(s_{t}, a_{t}, g\right)=\mathbb{E}\left[R_{t} \mid s_{t}, a_{t}, g\right]$

      q函数现在不仅依赖于状态动作对，还依赖于目标

3 Hindsight Experience Replay

3.1 A motivating example

  • a bit-flipping environment

    对二进制数进行位翻转

  • state : 是当前的二进制数， ，共有$2^N$种状态，每个状态的维度为N

  • action : 是从 个位置中挑选一个进行翻转

  • reward: 如果每次翻转后没有达到目标状态，那么reward就是-1。

  • 当 时，很多强化学习算法都不会得到很好的结果，因为在探索中除了-1，没有遇到过其他的reward。

  • 使用shaped reward可以在一定程度上解决这个问题，比如 ，但是对于其他复杂的任务并不能很简单的应用shaped reward

  • 但是如果使用HER的方法，就可以得到较好的结果

  • $s_{1}, \ldots, s_{T}$ and a goal $g \neq s_{1}, \ldots, s_{T}$
    用不同的目标重新审视这条轨迹

    虽然这个轨迹可能不能帮助我们学习如何实现状态g，但它肯定会告诉我们一些关于如何实现状态$s_T$的东西

  • 简单来说，每个任务都有自己的“真正目标”，但是在进行rollout的时候，并不是每个轨迹都能达成“真正目标”，对于那些没有走到目标重点的轨迹，为这些轨迹设置一个可完成“虚拟目标”，然后利用这个“虚拟目标”给出轨迹的reward，并使用强化学习算法进行训练。这和人类的学习能力其实是比较像的

  • $s_t$ 代替 g 放到 replay buffer
    至少有一半的回放轨迹包含不同于−1的奖励，学习变得更加简单

试图从初始状态 S 到达目标状态 G, 以某个状态 S’ 结束

$\left\{\left(S_{0}, G, a_{0}, r_{0}, S_{1}\right),\left(S_{1}, G, a_{1}, r_{1}, S_{2}\right), \ldots,\left(S_{n}, G, a_{n}, r_{n}, S^{\prime}\right)\right\}$

HER 中的想法是想象我们的目标实际上一直是 S’， agent成功地达到了目标并因此获得了积极的奖励。

除了缓存真实轨迹，还缓存了以下轨迹

$\left\{\left(S_{0}, G, a_{0}, r_{0}, S_{1}\right),\left(S_{1}, G, a_{1}, r_{1}, S_{2}\right), \ldots,\left(S_{n}, G, a_{n}, r_{n}, S^{\prime}\right)\right\}$

3.2 Multi-goal RL

  • training an agent to perform multiple tasks can be easier than training it to perform only one ta

  • HER gives improved performance even on tasks where we really care about a specific goal, so long as we can provide other goals for the purpose of training.

4 Experiments

  • Teste HER on several robotic manipulation tasks in which the agent must achieve different goals from initial states

    Agent receives a reward if it has completed the task in time, and no reward if it has not

reward shaping

domain-agnostic reward shaping does not work well

  • There is a huge discrepancy between what we optimize (i.e. a shaped reward function) and the success condition (i.e.is the object within some radius from the goal at the end of the episode);

  • Shaped rewards penalize for inappropriate behaviour (e.g. moving the box in a wrong direction) which may hinder exploration.

    strategies for choosing goals

  • final

    the ones corresponding to the final state of the environment

  • future
    replay with k random states which come from the same episode as the transition being replayed and were observed after it

  • random
    replay with k random states encountered so far in the whole training procedure
  • episode
    replay with k random states coming from the same episode as the transition being replayed
