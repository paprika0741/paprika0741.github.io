Deep Reinforcement Learning with Double Q-learning

Abstract

Problem of Q-learning：

  • overestimations can occur when the action values are inaccurate, irrespective

    of the source of approximation error

  • whether, in practice, such overestimations are common？

  • whether they harm performance ?

  • whether they can generally be prevented ?

Double DQN：

  • more accurate value estimates

Background：

Q-learning

learn estimates for the optimal value of each action

$Q_{\pi}(s, a) \equiv \mathbb{E}\left[R_{1}+\gamma R_{2}+\ldots \mid S_{0}=s, A_{0}=a, \pi\right]$

optimal value： $Q_{*}(s, a)=\max _{\pi} Q_{\pi}(s, a)$

$Y_{t}^{\mathrm{Q}} \equiv R_{t+1}+\gamma \max _{a} Q\left(S_{t+1}, a ; \boldsymbol{\theta}_{t}\right)$

$\boldsymbol{\theta}_{t+1}=\boldsymbol{\theta}_{t}+\alpha\left(Y_{t}^{\mathrm{Q}}-Q\left(S_{t}, A_{t} ; \boldsymbol{\theta}_{t}\right)\right) \nabla_{\boldsymbol{\theta}_{t}} Q\left(S_{t}, A_{t} ; \boldsymbol{\theta}_{t}\right)$

更新类似于随机梯度下降，更新当前值接近target $Y_t^Q$

DQN：

  • $Q(s, \cdot ; \boldsymbol{\theta})$

  • experience replay

  • target network $Q(s, \cdot ; \boldsymbol{\theta}^-)$

    its parameters are copied every $\tau$ steps from the online network

  • $Y_{t}^{\mathrm{DQN}} \equiv R_{t+1}+\gamma \max _{a} Q\left(S_{t+1}, a ; \boldsymbol{\theta}_{t}^{-}\right)$

​

  • standard Q-learning and DQN: uses the same network both to select and to evaluate an action

Double Q-learning

  • main idea: decouple the selection from the evaluation

  • select action: in the argmax online weight $\theta_t$

  • evaluate the value of this policy：$\theta_t’$

​

The maximization

  • Add zero-mean random noise to $x_{1}, \cdots, x_{n}$ and obtain $Q_{1}, \cdots, Q_{n}$.
  • The zero-mean noise does not affect the mean:
  • The zero-mean noise increases the maximum:
  • The zero-mean noise decreases the minimum:
  • True action-values: $x\left(a_{1}\right), \cdots, x\left(a_{n}\right)$.
  • Noisy estimations made by DQN: $Q\left(s, a_{1} ; \mathbf{w}\right), \cdots, Q\left(s, a_{n} ; \mathbf{w}\right)$.
  • Suppose the estimation is unbiased:
  • $q=\max _{a} Q(s, a ; \mathbf{w})$, is typically an overestimation:
  • Q-learning’s overestimations there indeed increase with the number of actions

  • function approximation

  • purple lines: true values

  • green dot: an approximation for a single action

  • green line: polynomial fitting

  • black lines: max

  • The maximum is often higher than the ground truth

  • Orange line is almost always positive, indicating an upward bias.

  • Double Q-learning on average much closer to zero
    Double Q-learning indeed can successfully reduce the overoptimism

  • Top and middle rows： overestimations are not an artifact of a specific true value function.

  • Middle and bottom rows：flexibility of the function approximation

    • d=6 insufficiently flexible，the estimates are even incorrect for some of the sampled states

    This is important because flexible parametric function approximators are often employed in reinforcement learning

  • Overestimations occur even when assuming we have samples of the true action value at certain states.

Bootstrapping

​

  • In RL, bootstrapping means “using an estimated value in the update step for the same kind of estimated value”.

  • Bootstrapping propagates the overestimation.

Why is overestimation harmful?

  • Learning Target: learn good policies for sequential decision problems, by optimizing a cumulative future reward signal.

  • Overoptimistic value estimates are not necessarily a problem in and of themselves.
    If all values would be uniformly higher then the relative action preferences are preserved and we would not expect the resulting policy to be any worse

  • sometimes it is good to be optimistic: optimism in the face of uncertainty is a well-known exploration technique

  • Overestimation combined with bootstrapping then has the pernicious effect of propagating the wrong relative information about which states are more valuable than others, directly affecting the quality of the learned policies

Double DQN

  • Double DQN decouples selection from evaluation.

  • $Y_{t}^{\text {DoubleDQN }} \equiv R_{t+1}+\gamma Q\left(S_{t+1}, \underset{a}{\operatorname{argmax}} Q\left(S_{t+1}, a ; \boldsymbol{\theta}_{t}\right), \boldsymbol{\theta}_{t}^{-}\right)$

​

  • Double DQN alleviates overestimation:

Empirical results

  • Double DQN improves over DQN both in terms of value accuracy and in terms of policy quality.

  • The top and middle rows show value estimates by DQN (orange) and Double DQN (blue) on six Atari games

    • DQN is consistently and sometimes vastly overoptimistic about the value of the current greedy policy

  • The bottom row shows the detrimental effect of this on the score achieved by the scores drop when the overestimations begin.

  • Learning with Double DQN is much more stable.

  • Overestimations are harming the quality of the resulting policies.

Quality of the learned policies

  • Overoptimism does not always adversely affect the quality of the learned policy.

  • Reducing overestimations can significantly benefit the stability of learning
