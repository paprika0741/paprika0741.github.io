自然语言处理(2)： N-gram Language Model

N-gram Language Model

Probabilistic Language Models

统计的语言模型

goal: assign a probability to a sentence　计算句子的概率

  • Machine Translation
    source language 翻译成target language，多个可能结果时，选择出现概率最大的
  • Spell Correction
    计算整个句子的概率，正确的词组出现概率远高于错误的词组
  • Speech Recognition
    语音识别选择概率大的作为识别的结果

概率越高，越符合人类的语言习惯

Probabilistic Language Modeling

Goal: compute the probability of a sentence or sequence of words:

计算句子的概率

$P(W) = P(w_1 ,w_2 ,w_3 ,w_4 ,w_5 … w_n)$

条件概率
计算下一个单词的概率
$P(W) = P( w_n|w_1 ,w_2 ,w_3 ,w_4 ,w_5 … w_{n-1})$

Chain Rule

$\mathrm{P}\left(\mathrm{x}_{1}, \mathrm{x}_{2}, \mathrm{x}_{3}, \ldots, \mathrm{x}_{\mathrm{n}}\right)=\mathrm{P}\left(\mathrm{x}_{1}\right) \mathrm{P}\left(\mathrm{x}_{2} \mid \mathrm{x}_{1}\right) \mathrm{P}\left(\mathrm{x}_{3} \mid \mathrm{x}_{1}, \mathrm{x}_{2}\right) \ldots \mathrm{P}\left(\mathrm{x}_{\mathrm{n}} \mid \mathrm{x}_{1}, \ldots, \mathrm{x}_{\mathrm{n}-1}\right)$

$P\left(w_{1} w_{2} \cdots w_{n}\right)=\prod_{i} P\left(w_{i} \mid w_{1} w_{2} \cdots w_{i-1}\right)$

​

problem：有些句子长 出现概率很低

solution：Markov Assumption

$P\left(w_{i} \mid w_{1} w_{2} \cdots w_{i-1}\right) \approx P\left(w_{i} \mid w_{i-k} \cdots w_{i-1}\right)$

只依赖于前面k个词语

Unigram model

一元文法

$P\left(w_{1} w_{2} \cdots w_{n}\right)=\prod_{i} P\left(w_{i}\right)$

每个词语都是独立的，没有考虑语词语间的关系，表达的信息非常有限

Bigram model

二元文法
每个词语只依赖于前面一个词语
$P\left(w_{i} \mid w_{1} w_{2} \cdots w_{i-1}\right) \approx P\left(w_{i} \mid w_{i-1}\right)$

N-gram models

problem: language has long-distance dependencies:

​

But we can often get away with N-gram models

Estimating N-gram Probabilities

极大似然估计 maximum likelihood estimate

$\begin{array}{l}
P\left(w_{i} \mid w_{i-1}\right)=\frac{\operatorname{count}\left(w_{i-1}, w_{i}\right)}{\operatorname{count}\left(w_{i-1}\right)} \\
P\left(w_{i} \mid w_{i-1}\right)=\frac{c\left(w_{i-1}, w_{i}\right)}{c\left(w_{i-1}\right)}
\end{array}$

训练过程
在预料库里计算次数，然后计算条件概率

Practical Issues

log place：转换到log空间计算

  • Avoid underflow 下溢
    (很多个小概率相乘，可能出现很接近0的浮点数，计算机表示可能表示成0，error)

  • adding is faster than multiplying
    $log(p_1p_2p_3*p_4) = log_{p_1}+log{p_2}+log{p_3}+log_{p_4}$

Evaluation and Perplexity

训练得到参数：每一项的条件概率

测试集上测试句子的概率

evaluation metric: 正确句子概率高 错误概率低

Extrinsic evaluation：

将模型运用到外部的任务上，间接的评估
缺点：Time-consuming 外部的任务时间久

intrinsic evaluation:

perplexity 困惑度：
the inverse probability of the test set, normalized by the number of words

用模型预测一个新的句子(正确的句子)的概率， 概率高则学的成功
The best language model is one that best predicts an unseen test set

困惑度越小越好
Minimizing perplexity is the same as maximizing probability

normalize：句子长度可能不一样 开n次方

$\begin{aligned}
P P(W) &=P\left(w_{1} w_{2} \ldots w_{N}\right)^{-\frac{1}{N}} \\
&=\sqrt[N]{\frac{1}{P\left(w_{1} w_{2} \ldots w_{N}\right)}}
\end{aligned}$

$Chain rule: \quad \operatorname{PP}(W)=\sqrt[N]{\prod_{i=1}^{N} \frac{1}{P\left(w_{i} \mid w_{1} \ldots w_{i-1}\right)}} $

For bigrams: $\quad \operatorname{PP}(W)=\sqrt[N]{\prod_{i=1}^{N} \frac{1}{P\left(w_{i} \mid w_{i-1}\right)}} $

Generalization and Zeros

  • 泛化

  • 如果其中某一项条件概率为0，所有的都会为0.训练集数量有限，不能涵盖所有语言的统计结果，完全可能训练集上没有出现的词组出现在测试集，整个句子概率为0.

    概率为0，无法计算perplexity （can’t divide by 0）

Smoothing: Laplace Smoothing

Add-one estimation

每一项加一
$MLE estimate: P_{M L E}\left(w_{i} \mid w_{i-1}\right)=\frac{c\left(w_{i-1}, w_{i}\right)}{c\left(w_{i-1}\right)} \\
Add-1 estimate: P_{A d d-1}\left(w_{i} \mid w_{i-1}\right)=\frac{c\left(w_{i-1}, w_{i}\right)+1}{c\left(w_{i-1}\right)+V} $

就不会出现0的概率
