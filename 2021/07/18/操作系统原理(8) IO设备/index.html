I/O设备

按使用特性分类：
人机交互类外部设备 计算机用户之间交互
打印机 显示器 鼠标		 字节 慢
存储设备 存储程序和数据
磁盘 磁带 光盘		 块 快
网络通信设备 与远程设备通信
网络接口 调制解调器		 中间

按传输速率分类
低速设备 每秒几个/几百个字节 鼠标 键盘
中速设备 数千/数万 打印机 激光打印机
高速设备 数百个千字节/千兆字节 磁带机 磁盘机 光盘

按信息交换单位分类

单位
块设备 块 有结构设备，传输速率比较高，可寻址（可随机读/写一块） 磁盘
字符设备 字符 无结果类型，速率低，不可寻址，输入输出常采用中断驱动方式 打印机 交互式终端

组成

机械部分：设备本身（物理装置）
电子部分：设备控制器/适配器
OS总是与控制器打交道而不是设备本身（中介）

I/O控制器

接受和识别CPU发出的命令 — 控制寄存器存放命令和参数
向CPU报告设备的状态 — 状态寄存器，CPU读取内容判断设备的状态
数据交换 — 数据寄存器，暂时存放数据，完成数据交换
地址识别 — 判断CPU要读/写哪个地址

​

​

I/O控制方式

程序直接控制

​

①CPU发出指令，设备启动，状态寄存器1
②CPU不断轮询设备是否准备好数据
③准备好数据后，数据在数据寄存器，状态寄存器修改 为0
④CPU从数据寄存器中读出数据到CPU的寄存器中
⑤ 数据从CPU寄存器到内存

CPU干预频率：频繁，I/O开始结束都需要CPU；期间一直轮询检查

数据传送单位：字

数据流向：
读： I/O -> CPU -> 内存
写： 内存-> CPU -> I/O
每个字都需要CPU

优点：实现简单
缺点：CPU和I/O只能串行；CPU轮询检查，长期处于忙等，CPU利用率低

中断驱动

​

注意：
①CPU每条指令末尾检查中断

②中断处理需要保存现场 恢复现场 （开销）

CPU干预频率：频繁，I/O开始结束都需要CPU；期间一直CPU 切换到别的检查

数据传送单位：字

数据流向：
读： I/O -> CPU -> 内存
写： 内存-> CPU -> I/O
每个字都需要CPU

优点：
与程序控制相比，不需要轮询，由中断信号通告传输完成。CPU和I/O可以并行

缺点：
每个字传输都需要CPU
频繁中断开销大

DMA方式

Direct Memory Access 直接存储器存取

  • 许多计算机为了避免用 IO 增加 CPU 的负担，将一部分任务下放给一个专用处理器，称之为直接内存访问控制器。
  • 主要用于块设备

改进：
① 块为单位
② 数据流向不需要经过CPU，设备—内存
③ 只有块开始/快结束 才需要CPU干预

​

​

（DMA仍是字读入，全部读完，写内存，中断通知CPU）

CPU干预频率：块开始/结束才需要CPU

数据传送单位：块/ 多个连续的块

数据流向：
读： I/O -> 内存
写： 内存-> I/O
每个字都需要CPU

优点：
块为单位，CPU介入频率低，并行性提高

缺点：
一个/多少连续的数据块

通道控制方式

识别执行一系列通道指令

I/O系统层次结构

​

用户层软件 库过程中的I/O系统调用
输入，如gets
输出，如printf
SPOOLing系统：用共享设备(如磁盘)模拟独占设备(如打印机) 打印精灵 网络传输精灵

I/O调度

确定处理I/O请求的顺序

设备保护

看成一种特殊的文件
每个设备对应FCB

假脱机计数

脱机：脱离主机控制 输入输出操作

脱机技术引入：批处理阶段，为了缓和CPU高速与I/O低速的矛盾引入脱机输入/输出

用专门的外围控制机，在I/O设备与磁盘直接传输数据

SPOOL（Simultaneous Peripheral Operations On Line，外部设备同时联机操作/假脱机）技术也称为虚拟设备技术。 用软件的方式模拟脱机技术

​

​

​ 用户进程和输入/输出进程需要并发，需要多道程序 技术支持

​ 缓冲区都是在内存

​ 优点：
​ 高速虚拟I/O操作
​ 实现对独享设备的共享 （共享打印机）

​

I/O缓冲

缓冲技术可提高外设利用率，尽可能使外设处于忙状态

限制：进程的I/O请求不能超过外设的处理能力

  • 解决CPU与I/O设备速度不匹配的矛盾
  • 减少对CPU的中断频率
  • 解决基本数据单元大小不匹配的问题
  • 提高CPU与I/O设备之间并行性

内存开辟缓冲区：
非空，不能传入，只能传出
空，必须充满，才能传出

单缓冲

一个

双缓冲

两个
OS清空或填充一个缓冲区时，进程可以传送数据到另外一个缓冲区（或从另外一个缓冲区取数据）

循环缓存

多个，每个缓冲区是循环缓冲区的一个单元
使I/O操作能跟上（可能爆发式地执行大量I/O的）进程的速度
有限缓冲区的生产者/消费者模型
