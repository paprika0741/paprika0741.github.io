文件管理

字段/域（field）：
基本数据单元（数据类型、长度，如姓名、年龄）

记录（record）：
一组相关的域（如雇员的信息）的集合
变长/定长
访问结构化文件的逻辑单元

文件（file）：
相似记录的集合（通过名字访问的实体）
以计算机硬盘为载体存储在计算机上信息的集合
系统运行时，进程为基本单位；用户输入输出时，文件为基本单位
数据库（database）：
相关数据的集合（由若干类型的文件组成）,本质特征是数据元素之间存在明确的关系

文件管理系统:

一组系统软件,为使用文件的用户和应用程序提供服务

功能:

上层用户提供的服务

文件命名：作为文件标识
访问控制：并发访问和用户权限
文件结构管理：划分记录，顺序，索引
目录管理：用于文件访问和控制的信息，不包括文件内容
文件操作：文件的创建、打开、读写和关闭
文件存取：用于完成文件的修改、追加、搜索等操作限额(quota)：限制每个用户能够建立的文件数目、占用外存空间大小等
审计(auditing)：记录对指定文件的使用信息（如访问时间和用户等），保存在日志中

对于用户：
实现对文件的基本操作、按名存储和查找文件、文件共享和文件保护功能

对于操作系统本身：
管理与磁盘的信息交换 文件逻辑结构和物理结构的交换
组织文件再磁盘上的存放 采取好的存放方式和磁盘调度提高整个系统的性能

文件组织 file organization文件中记录的逻辑结构

文件中记录的逻辑结构
用户观点看到文件的组织形式

设计要求

访问性能：访问快速（便于检索）、易于修改
存储性能：节省空间、维护简单（向物理存储转换方便）、可靠

无结构文件：

字符流/流式文件。
由于没有结构，访问只能穷举搜索，故对大多数文件不适用
适用于对基本信息单位操作不多的文件，源程序文件 目标代码文件

有结构文件：

由一组相似记录组成，记录式文件。
每条记录由若干数据项组成。

顺序文件

记录一个接一个顺序排列（逻辑上），所有记录长度相同，相同数量 长度固定的域 的 顺序组成

​

关键域：每条记录的第一个域，唯一地表示记录，记录按照关键域存储

物理实现：顺序存储 链式存储

优点：实现简单
缺点：查询和更新性能差（必须顺序搜索，插入新纪录困难）

索引顺序文件

保留顺序文件的关键特征：记录按照关键域顺序组织
增加特征：增加随机索引（支持随机访问）和溢出文件
索引文件：记录关键字段和指向文件的指针
溢出区(overflow):处理新纪录
主文件每个记录增加一个附加字段，保存指向溢出区 新纪录的指针，初始化为空。
新纪录加到溢出区

将记录分组，每组对应一个索引表项(每组第一个记录在索引表里)。
检索时，先顺序查找索引表，找到分组，再顺序查找分组

优点：
提高存取速度，保留文件顺序存储的特性
例如：100万条记录的顺序文件，查找某个特定关键域值，平均访问50万次。如果1000项的索引表，先平均查500次，再平均查500次。
开销从500000降到了1000

缺点：增加了存储空间

索引文件

顺序文件的限制：基于非关键域的其他属性查找

多索引结构：成为查找条件的每个域都可能有一个索引

抛弃顺序性和关键域，采用多索引结构，可使用变长记录。

完全索引：包含主文件每条记录的索引项。
为了易于查找，索引表自身组织成顺序文件

部分索引:只包含有感兴趣域的记录的索引项

  • 主要用于强调速度但很少会处理文件的所有数据的场合

​

直接文件或散列文件
  • 保留关键域、抛弃排列顺序，使用基于关键域的散列，定长记录，快速访问已知单记录块
  • 常用于要求快速访问记录的情形（如目录、价格表等）

文件目录

文件控制块 （FCB）/文件说明

文件名到文件内容/数据的映射

基本信息： 文件名
文件类型：有无结构(流式/记录式)/内容（二进制/文本）/用途/属性
文件结构
地址信息 卷：存储文件的设备
起始地址：辅存中的起始物理地址
文件长度
访问控制信息 所有者 、访问权限、许可的行为
使用信息 数据创建、创建者身份、最近访问、最近修改..

文件目录

FCB的有序集合，一个FCB就是一个文件目录项
创建一个新的文件，系统分配一个新的FCB并存放在文件目录中
主要功能：从文件名到文件内容的映射

文件访问的步骤：

  • 目录检索（根据用户给出的文件名，按名寻找目录项）
  • 文件寻址（根据目录项中的文件物理地址等信息，计算出文件的任意记录或字节在存取介质上的地址）

目录操作：

搜索：用户使用文件时，根据文件名搜索目录，找到文件对应目录项
创建：增加目录项
删除：删除相应目录项
显式目录：
修改目录：

目录结构：

目录的组织结构，设计目标主要是提高检索速率

单极目录结构：

整个系统中只建立一张目录表，每个文件占一个目录项，实现“按名存取”

优点：

  • 结构简单，易实现

缺点：

  • 查找速度慢
  • 不允许文件重名
  • 不便于文件共享，对于多用户操作系统不适用

两级目录结构：

适用于多用户系统，各用户可有自己的专用目录

​
主文件目录 （Master File Directory):
每个用户对应一个目录

用户文件目录(User File Directory):
该用户的文件FCB组成

优点：

  • 降低了平均检索时间
  • 允许不同用户文件重名， 部分克服了命名冲突问题

缺点:

  • 缺乏灵活性，不利用户对文件的分类管理

多级目录结构/ 树形目录结构

层次结构，每个用户可在其用户目录下自主创建若干层次的子目录(树)

​

目录名：可以修改
目录树：中间结点是目录，叶结点是目录或文件
目录的上下级关系：

​ 当前目录 (current directory, working directory工 作目录)，一般用单个句点表示：.
​ 父目录 (parent directory) ，一般用双句点表示：..
​ 子目录 (subdirectory)
​ 根目录(root directory) ，一般用单斜线表示
​ 路径(path)：每个目录或文件，可以由根目录开始依 次经由的各级目录名，加上最终的目录名或文件名来 表示。一般用斜线分隔。

优点：

  • 层次结构清晰，便于管理和保护，适用于较大的文件系统管理
  • 解决了命名冲突问题

缺点：

  • 目录级别太多时，会增加路径检索时间（目录文件存放在外存）
  • 不便于文件共享

改进的多级目录

其实在查找各级目录地过程中，只需要用到”文件名”这个信息，只有文件名匹配时，才需要读出文件地其他信息。

符号目录项(次部)：文件名、文件内部标识号
基本目录项BFCB(主部)：其余文件说明信息

​

索引结点：除文件名之外的其他信息
目录中只包含文件名和指向索引结点的指针。

好处：

翻车：

① 计算分解前存储所有FCB需要的盘块$\frac{ 128\times48}{512} = 12$

实际上一块512 = 48 x 10 + 32， 32是内部碎片，不能用的
要先计算一块可以存储的FCB，$\lfloor { \frac{512}{48}}\rfloor = 10$
所有FCB存储需要$\lceil \frac{128}{10} \rceil = 13$

②计算分解后
首先查符号目录表，再查基本目录文件，需要+1

记录组块

文件的逻辑单位是记录(record)，外存的 I/O 操作单位是块(block)，为了进行 I/O 操作，需要把文件的记录组织成块

块定长：简化I/O，内存缓冲分配和外存块组织

块大小：
块越大：一次I/O传送记录多。利于顺序处理/搜索，若随机访问文件记录，会有不必要的传输
块越小：要求内存缓冲越大，缓冲管理越难

记录组块的方法

定长组块 记录长度固定，每个块中存放的记录数目也固定 常用但有内部碎片
变长跨越式 记录长度可变，一个记录允许存放在不同块中 效率高但难实现
变长非跨越式 记录长度可变，一个记录只能存放在一个块中 浪费多

辅存管理

  • 文件分配

从文件在物理介质上的存放方式来研究文件，即讨论文件的物理结构

  • 空闲空间管理

跟踪可用空间

预分配与动态分配

  • 预分配（preallocation）

    文件创建时必须要声明文件的最大长度
    难以可靠估计文件可能的最大长度
    倾向于估大文件长度（导致空间浪费）
    适用于从不改变的文件

  • 动态分配（dynamic allocation）

    需要时才给文件分配空间

区块大小

分配给文件的区块大小/以多大的区块为分配单位
考虑因素:

  • 连续的空间可提高性能
  • 区块多会大家文件分配表的大小
  • 固定大小分区可简化空间的再分配
  • 可变大小/固定大小的小分区可以减少存储空间浪费

可变的大规模连续分区：
性能较好。大小可变避免浪费，文件分配表较小，但是空间很难再次利用

块：小的固定分区提高灵活性，但是文件分配表更大/复杂

文件物理结构（文件分配方式）

从文件在物理介质上的存放方式来研究文件（文件应该怎样存放在外存中），即讨论文件的物理结构

连续分配：

每个文件在磁盘上占有一组连续地块
(逻辑块号，块内地址)->(物理块号,块内地址)
只需要转换块号,块内地址不变
文件分配表表项: 起始块号和长度（实现地址映射）

步骤：
找到文件对应的目录项
查到起始块号和长度，
起始块号+逻辑块号 = 物理块号

​

如果需要aaa的第1块，查表得到起始是4，那么第1块的物理地址是4+1-1

优点

  • 实现简单（只需在文件分配表FAT中记录起始块号和长度）
  • 支持顺序访问和直接访问， 存取速度快
  • 顺序存取速度快，所需的磁盘寻道次数和寻道时间最少

缺点

  • 不方便文件扩展
  • 反复增删会产生外部碎片（需要重新分配和压缩）
  • 预分配，创建文件需要声明文件大小

链接分配:

一个文件的信息存放在若干不连续的物理块中
各块之间通过指针连接，前一个物理块指向下一个物理块

过程：
文件名，找到目录项，找到起始块号，要将这个块调入，才能知道下一个块的存放位置。
因此，访问第i块，前面的全部都要访问，只能顺序访问

​

优点

  • 消除了外部碎片,提高磁盘空间利用率
  • 有利于文件动态扩充

缺点

  • 不适于随机存取,只能顺序存取
  • 存取速度慢，更多的寻道次数和寻道时间

改进

（显式链接）

FAT：用于连接各文件物理块的指针显示存放在一块表中。
需要连续存储，并且表项大小相同（才能用索引查表）
整个磁盘设置一张（开机时读入内存并常驻内存,查询不需要磁盘）

​

步骤：

首先aaa起始块号2，找到下一块5，再找下一块0…

效果：
支持随机访问,但是文件分配表需要占一定的存储空间。

索引分配

文件的信息存放在若干不连续物理块中
系统为每个文件建立一个专用数据结构——索引表，并将这些块的块号存放在索引表中

​ （记录一一映射关系 ， 节约内存就放在外存里，只要记下索引块在哪，就能得到表，就能知道文件所有块在哪）

索引块:存放索引表的磁盘块
数据块:存放数据的磁盘块

转换:

目录中根据文件名找到索引块号
将索引块从外存读入内存
查找索引表第i号逻辑块对应得物理块

“aaa”的索引表在7号块，7号块是索引块，
读7号块，查索引表，得到文件每一块的物理块号

优点

  • 顺序存取 随机存取
  • 满足了文件动态增长、插入删除的要求，也能充分利用外存空间

缺点

  • 较多的寻道次数和寻道时间
  • 索引表本身带来了系统开销：内外存空间、存取时间（文件每一块都需要记录）

问题：如果索引表过大，一个块存不下
链接方案：多个索引块，链接起来。FCB中记录第一个索引块的块号
多层索引：多层的索引表。FCB记录顶级索引表的块号。
如果一块最多256各索引项。二级索引表。若要查 询1024号逻辑块。首先1024/256=4 ，查询第4项 得到块号，调入，
1024%256=2，查询这块的第二项，得到块号
K层索引结构，顶级索引表未调入内存。访问数据块需要K+1次读磁盘
混合索引：

连续 链接 索引
预分配 必需 可能 可能
区块大小 可变 固定块 固定块 可变
区块尺寸 大 小 小 中等

分配时间 中等 长 短 中等

文件共享

通过唯一的共享路径：
访问速度慢
多个目录中的不同文件名(文件别名):
访问速度快，但会影响文件系统的树状结构，适用于经常访问的文件共享，同时存在一定的限制

文件别名的实现方法：

硬链接

基于索引结点的方式（改进的多级目录结构）：

目录项分为两部分
符号目录项：文件名 + 索引结点指针
基本目录项（索引结点）：FCB其他信息 +链接计数

count(表示链接到这个结点的用户目录数)

当count=0，系统负责删除文件

存在问题：

记账或配额问题：
设B、C共享文件file，C是文件主，若C删除了file而B不删除，则file仍然存在，且仍然是C为file付账，占的也是C的配额

软链接：

基于符号链接的方式：

  • Link类型的特殊文件，内容是到另一个目录/文件路径的链接,不影响原文件,事实上各是一个文件
  • 根据路径一层层查找目录
  • 可以建立任意的别名关系，甚至原文件是在其他计算机上

缺点:

时间空间开销大

其他用户读共享文件时，需要根据文件路径逐个查找多级目录，直到找到文件的索引结点
访问文件开销变大，增加磁盘启动的频率
符号链的索引结点也要耗费一定的磁盘空间
