处理器调度

对处理器分配

类型

长程/作业/高级 决定哪些新建进程可进入系统准备执行 内存辅存之间
控制多道程序系统的并发程度
进程越多则各进程对CPU的使用百分比越小
中程/内存 决定交换哪些主存－辅存（内存-外存）进程 基于多道程序设计的管理需要
提高内存利用率，和系统吞吐量
短程/低级 决定下一个使用CPU的进程（dispatcher，分派程序）

​

长程调度为进程活动做准备
短程调度使进程正常活动起来
中程将暂时不能运行进程挂起

​

短程调度的时机

当前进程正常或异常终止（通过中断实现）
时钟或I/O中断
系统调用（通过软中断实现）
信号量操作（通过软中断实现）

模式

非剥夺式

只允许主动释放处理机（阻塞/完成）
优点：实现简单 系统开销小
缺点：不适合分时系统和大多数实时系统
(分时系统时间片剥夺)

剥夺式

当更为重要/紧迫的进程就绪，暂停当前进程，移交CPU使用权

优点：
避免单一进程长时间占处理机
提高吞吐率和响应效率
缺点：
系统开销大 （降低途径：硬件实现进程切换，或扩充主存以贮存大部分程序）

短进程调度目标

公平——确保每个进程都获得合理的CPU份额
效率——使CPU及其他系统资源尽量忙碌
响应时间（从提交到首次响应）——尽可能短
在交互式系统中尤为重要
周转时间Tr（turnaround time：从提交到结束）（也叫驻留时间，residence time）——尽可能短
吞吐量（单位时间内完成的进程数）——尽可能大
实时性——可以指定进程完成的最后期限

调度时机
先来先服务FCFS 不可剥夺 不会饥饿 实现简单，效率低；长作业有利，短作业不利
利于CPU繁忙型作业，不利于I/O型繁忙作业		 当前让出（完成/阻塞）
最短进程优先（SPN） 每次调度选取估计运行时间最短的进程 不可剥夺式 “长”进程可能饿死 预知或估计进程的执行时间 当前让出（完成/阻塞）
最短剩余优先（SRT） 剥夺式的SPN

新进程进入就绪队列时将引发重新调度
选取估计剩余时间最短的那个进程 | 剥夺式 | “长”进程可能饿死 | 估计执行时间 | ①当前让出（完成/阻塞）
②新进程 |
| 最高响应比优先（HRRN） | 每次调度选取响应比R最大的那个进程
w：等待CPU的时间
s：估计执行时间
R＝（w+s)/s = 周转/执行 | 非剥夺式 | 利短进程，但长进程也不会饿死 | 估计执行时间
计算R开销 | 当前让出（完成/阻塞） |
| 时间片轮转（RR） | 剥夺式的FCFS
把CPU划分成若干时间片,并且按顺序分配给就绪列队中的各个进程
当时间片用完而当前进程尚未结束时，系统剥夺该进程的CPU，将该进程排列到就绪队列的末尾，同时选择就绪队列头的进程运行 | 剥夺式 | | 利CPU密集进程，不利I/O密集进程 | ①当前让出（完成/阻塞）
②时间片到 |
| 最高优先级优先（HPF） | 每个进程被赋予一个优先级（priority），每次调度选取就绪队列中优先级最高的进程投入运行 | | | | |
| 多级队列反馈（MF/FB） | 关注进程已执行的时间，不用估计（剩余）执行时间
采用动态优先级机制 | | | | |

RR 时间片确定：

太短：切换开销大
太长：响应时间变长
一般比典型的一次交互过程时间略长
分时系统和事务处理系统中常用时间片轮转法

虚拟轮转：
解决轮转对I/O密集型进程的不公平性问题

阻塞解除的进程进入辅助队列
辅助队列中的进程比就绪队列的优先获得处理器

优先级确定方法：
静态：
进程创建时指定优先级，在进程运行时优先级保持不变

动态：
在进程创建时指定一个优先级，但在其生命周期内优先级可以动态变化（如等待时间长的优先级可提高，时间片过后的优先级可降低 ）

实现：
对应不同优先级采用多个就绪队列

多级队列反馈（MF/FB）

  • 多个优先级就绪队列，各个队列运行时间片可能不同（优先级$i$,时间片越小$2^i$）
  • 新的就绪进程进入最高优先级队列
  • 进程由于时间片用完被抢占而放弃CPU，下降一个优先级队列（最高－>最低）
  • 进程由于等待而放弃CPU后，进入等待队列，一旦等待的事件发生，则回到原来的就绪队列
    （可能对I/O密集型有利）
  • 在各优先级队列中采用FCFS（最低级队列中采用RR
  • 调度时机
    ①当前让出（完成/阻塞）当前进程优先级不变
    ②时间片到 发生抢占才会降级（时间片到不代表抢占）
    （如果当前不止一个进程就会时间片到就会抢占，优先级只是用来选择下一个。如果几个优先级队列，选高的那个，同一队列，FCFS）
